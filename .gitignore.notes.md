# Git Configuration Notes

## Files That Should Be Committed

The following files contain configuration for different environments and should be committed to the repository:

- `.env.prod` - Production environment variables
- `.env.test` - Test environment variables
- `docker-compose.yml` - Local development
- `docker-compose.prod.yml` - Production deployment
- `docker-compose.test.yml` - Test deployment
- `nginx/*.conf` - Nginx configurations for all environments

## Files That Should NOT Be Committed

- `.env` - Local development secrets
- `.env.local` - Local overrides
- `tamkin-backend/.env` - Backend local config
- `tamkin-frontend/.env` - Frontend local config
- `node_modules/` - Dependencies
- `dist/`, `.next/` - Build outputs
- `postgres-data*/` - Database volumes

## Important Notes

1. `.env.prod` and `.env.test` should contain non-sensitive defaults or placeholders
2. Actual production secrets should be managed through your deployment platform
3. For local development, copy `.env.prod` to `.env` and modify as needed
